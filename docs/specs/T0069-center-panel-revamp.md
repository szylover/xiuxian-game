# 设计文档：中央区域布局重构 — 场景视图

任务：T0069
日期：2026-04-03（v2 重设计）

## 概述

将中央区域从"操作按钮 + 日志长卷"改造为**场景视图（SceneView）**——玩家站在某个地点，看到周围的人、可去的方向、能做的事。日志改为可收起的侧边抽屉。整体效果类似经典 RPG 的主画面：你"身处"一个场景，而非盯着一张数据表。

---

## 现状分析

### 当前布局

│          │ │   占据 ~85% 高度  │ │ │炼制   │ │

│          │ │                  │ │ │成就   │ │

### 问题

1. **日志喧宾夺主**：GameLog 通过 `flex: 1` 占满中栏剩余空间，成为视觉焦点
2. **核心世界交互被边缘化**：地图（区域导航）和 NPC（社交对话）被塞在右栏浮动面板里，等于"藏了两级菜单"
3. **中央无场景感**：玩家无法直观感受到"我在哪、身边有谁、能去哪"
4. **操作按钮区太小**：ActionPanel 只占一小条，主要玩法入口不够显眼

---

## 设计方案

### 核心理念：场景视图

中央区域变成一个**场景画面**——玩家"站在"当前区域里，能看到：

- **我在哪** → 区域名称、描述、标签（危险/安全/加成）
- **身边有谁** → 当前区域的 NPC 卡片，标识谁有任务
- **能去哪** → 相邻区域（父/子/兄弟）的出口列表
- **能做什么** → 修炼/战斗/探索/休息 + 突破按钮
- **我的进度** → 修为进度条 + 瓶颈提示 + 任务追踪

### 各区块详细设计

#### 1. 区域标题栏 — SceneHeader

展示"你站在哪里"的沉浸感。

```
┌──────────────────────────────────────────┐
│  ═══  🏔️  灵 霄 山 脉  ═══              │
│  "灵气充沛之地，云雾缭绕，偶有妖兽出没"    │
│  ⚔️ 危险区域  ·  🔍 探索+20%             │
└──────────────────────────────────────────┘
```

- **区域名**：大号字体居中，左右加像素风装饰线 `═══`
- **描述文字**：一行斜体引用
- **标签**：从 `RegionDef` 的 `safeZone` / `combatBonus` / `explorationBonus` / `shopDiscount` 生成
- 背景可以用微妙的渐变色区分区域类型（安全区偏蓝绿，危险区偏暗红）

#### 2. NPC 区域 — SceneNpcs

展示当前区域的 NPC，可点击互动。

**有 NPC 时：**

```
┌──────────────────────────────────────────┐
│  此地之人                    [📋 人脉总览] │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │    👴    │ │    💃    │ │    🧙    │ │
│  │ 青云长老  │ │ 月灵仙子  │ │ 散修张三  │ │
│  │ ♥♥♥♡♡   │ │ ❗有任务   │ │ ♥♡♡♡♡   │ │
│  └──────────┘ └──────────┘ └──────────┘ │
└──────────────────────────────────────────┘
```

- NPC 卡片横向排列，超过 4 个时横向可滚动
- 每张卡片显示：NPC emoji/头像、名字、好感度心形、任务标记（❗可接 / ✅可交付）
- 点击卡片 → 打开 `NpcDetailModal`（复用现有组件）
- 右上角 **[📋 人脉总览]** 按钮 → 打开 NpcPanel 浮动面板（contacts tab），查看所有已邂逅 NPC

**无 NPC 时：**

```
┌──────────────────────────────────────────┐
│  此地之人                    [📋 人脉总览] │
│        此地空无一人，只有风声呼啸           │
└──────────────────────────────────────────┘
```

#### 3. 出口导航 — SceneExits

展示从当前位置可以去哪里，像文字冒险游戏的方向选择。

```
┌──────────────────────────────────────────┐
│  可去之处                      [🗺️ 全图]  │
│                                          │
│  → 🏘️ 凡人集市     5⚡        [前往]     │
│  → 🌲 幽暗密林     8⚡        [前往]     │
│  → ⬆ 灵霄山脉·外围  3⚡        [前往]     │
│  → 🗻 太虚峰顶     🔒 需筑基期            │
└──────────────────────────────────────────┘
```

- **出口来源**：根据区域树自动计算相邻区域
  - 父区域（⬆ 标记，表示"返回"）
  - 子区域（⬇ 更深入的地点）
  - 兄弟区域（同级别的其他地点）
- 每个出口显示：emoji + 名称 + 精力消耗 + [前往] 按钮
- 未解锁的区域：灰色 + 🔒 + 解锁条件文字
- 出口列表最多显示 **5 个**，超过时收起为 `还有 N 个区域...` + 展开按钮
- **[🗺️ 全图]** 按钮 → 打开 MapPanel 浮动面板，查看完整区域树
- 点击 [前往] → 调用 `onTravel(regionId)`，场景刷新为新区域

**出口计算伪代码：**

```
getSceneExits(currentRegionId, allRegions):
  current = find(currentRegionId)
  exits = []

  // 父区域（返回上一级）
  if current.parentId && !parent.isContainer:
    exits.push({ ...parent, direction: 'up' })

  // 子区域（探索更深处）
  children = allRegions.filter(r => r.parentId === currentRegionId && !r.isContainer)
  exits.push(...children as 'down')

  // 兄弟区域（同级其他地点）
  if current.parentId:
    siblings = allRegions.filter(r =>
      r.parentId === current.parentId &&
      r.id !== currentRegionId &&
      !r.isContainer
    )
    exits.push(...siblings as 'sibling')

  return exits  // 带 access check + travel cost
```

#### 4. 修炼进度 — CultivationProgress

紧凑的修为进度显示，比左栏的迷你条更醒目，但不喧宾夺主。

```
┌──────────────────────────────────────────┐
│  练气九层 ▸ 筑基初期                      │
│  [████████████████░░░░] 82%  12400/15000 │
│  🔥 铜皮 ▸ 铁骨  [████████░░░░] 65%      │  ← 仅体修激活时显示
└──────────────────────────────────────────┘
```

- 一行显示当前境界 → 下一境界
- 进度条用渐变色，气修绿色系，体修橙色系
- 附带数值（当前/需要）
- 体修行仅在 `player.bodyRealmIndex > 0` 时显示

#### 5. 操作按钮 — ActionPanel（精简版）

沿用现有 ActionPanel，但移除瓶颈提示条（迁至底部状态栏），按钮布局调整为更宽敞的网格。

```
┌──────────────────────────────────────────┐
│  [🧘 修炼]   [⚔️ 战斗]   [🔍 探索]      │
│  [💤 休息]   [🎆 气修突破 → 筑基 (78%)]  │
│              [🔥 体修突破 → 铁骨]         │  ← 条件满足时显示
└──────────────────────────────────────────┘
```

- 4 个核心按钮排成一行或 2×2 网格（取决于宽度）
- 突破按钮占整行，更醒目
- 安全区域时战斗按钮显示 🛡️ 并禁用（现有逻辑不变）
- 瓶颈提示条从此处移除

#### 6. 底部状态栏 — SceneFooter

场景底部的信息条，汇总关键状态 + 日志入口。

```
┌──────────────────────────────────────────┐
│ ⚠️ 心魔瓶颈 [████░░] 40%   📋 寻灵草 3/5│
│                                  [📜 日志]│
└──────────────────────────────────────────┘
```

- **瓶颈提示**（左侧）：若有激活瓶颈，显示名称 + 坚韧修炼进度条。点击弹出瓶颈详情弹窗（复用现有逻辑）
- **任务追踪**（中间）：若有追踪中的任务，显示当前目标摘要（一行）
- **日志按钮**（右下角）：点击打开/关闭日志抽屉。有新日志时可显示未读数量徽章
- 无瓶颈时该行只显示任务追踪 + 日志按钮
- 无任务也无瓶颈时该行只显示日志按钮

### 日志抽屉 — LogDrawer

GameLog 从中央常驻改为**右侧滑出抽屉**。

```
  场景视图保持不变
         ┌──────────────────────────┐
         │ 📜 日志              [✕] │
         │ ┌──────────────────────┐ │
         │ │ [全部][战斗][探索]...│ │  ← 筛选标签
         │ ├──────────────────────┤ │
         │ │ ▼ 第3年              │ │
         │ │   ▼ 三月 (12) [当前] │ │
         │ │     获得灵草×2       │ │
         │ │     击败妖兽...      │ │
         │ │   ▶ 二月 (8)        │ │
         │ │ ▶ 第2年              │ │
         │ └──────────────────────┘ │
         └──────────────────────────┘
```

- **触发**：点击场景底部的 [📜 日志] 按钮
- **位置**：`position: fixed; right: 0; top: 0; height: 100vh`
- **宽度**：380px（和右侧浮动面板类似宽度）
- **层级**：`z-index: 300`（高于浮动面板的 200）
- **动画**：`transform: translateX(100%) → translateX(0)`，0.3s ease
- **关闭**：点击 ✕ 按钮、点击左侧半透明遮罩、或按 ESC
- **内部**：完整复用现有 GameLog 组件（分类筛选、年/月折叠、时间线）
- **遮罩**：左侧 `rgba(0, 0, 0, 0.3)`，点击关闭抽屉

## 数据结构

无需新增 Player 属性或类型定义。仅涉及 UI 状态：

```ts
// App.tsx 新增状态
const [logDrawerOpen, setLogDrawerOpen] = useState(false);
```

出口计算需要新增工具函数（纯逻辑，不修改数据结构）：

```ts
// game/map.ts 新增
interface SceneExit {
  region: RegionDef;
  direction: 'up' | 'down' | 'sibling';
  travelCost: number;
  canEnter: boolean;
  lockReason?: string;
}

function getSceneExits(player: Player): SceneExit[]
```

### getSceneExits 逻辑

```
function getSceneExits(player):
  current = getCurrentRegion(player)
  allRegions = getAllRegions()
  exits = []

  // 1. 父区域（⬆ 返回上级）
  if current.parentId:
    parent = find(current.parentId)
    if parent && !parent.isContainer:
      exits.push({ region: parent, direction: 'up' })

  // 2. 子区域（⬇ 探索更深处）
  children = allRegions
    .filter(r => r.parentId === current.id && !r.isContainer)
    .sortBy(minRealm)
  exits.push(...children as direction: 'down')

  // 3. 兄弟区域（同级其他地点）
  if current.parentId:
    siblings = allRegions
      .filter(r =>
        r.parentId === current.parentId &&
        r.id !== current.id &&
        !r.isContainer
      )
      .sortBy(minRealm)
    exits.push(...siblings as direction: 'sibling')

  // 每个出口附带 access check + travel cost
  return exits.map(e => ({
    ...e,
    travelCost: calcTravelCost(player, e.region),
    canEnter: checkRegionAccess(player, e.region.id).canEnter,
    lockReason: checkRegionAccess(player, e.region.id).reason,
  }))
```
