# Roadmap — 修仙小游戏

> 从最小可玩原型到完整修仙世界的阶段规划。  
> 每个 Milestone 包含若干 Stage，按顺序实现。

---

## Milestone A: 核心循环（最小可玩） ⬜

> 目标：玩家能修炼、战斗、突破、存档。一个人玩起来就停不下来的核心循环。

| Stage | 名称 | 说明 | 状态 |
|-------|------|------|------|
| A-1 | 属性系统 | 四类属性（基础/战斗/先天/资质）完整实现到 `Player` 类 | ⬜ |
| A-2 | 修炼系统 | 修为获取受悟性/灵根/心情乘数影响，精力消耗限制操作次数 | ⬜ |
| A-3 | 战斗系统 v2 | 按设计文档的伤害公式重写（减防→暴击→抗性→闪避），先手规则 | ⬜ |
| A-4 | 境界突破 | 突破判定（修为门槛 + 成功率受悟性/幸运影响），失败有惩罚 | ⬜ |
| A-5 | 寿命系统 | 操作推进时间，寿元耗尽 → 游戏结束，境界提升增加寿限 | ⬜ |
| A-6 | 状态面板 v2 | 四栏属性展示（基础/战斗/先天/灵根），进度条可视化 | ⬜ |

## Milestone B: 随机事件引擎 ⬜

> 目标：探索充满惊喜，幸运属性真正有意义。

| Stage | 名称 | 说明 | 状态 |
|-------|------|------|------|
| B-1 | 事件引擎重构 | `luck` 驱动的加权随机系统，事件分好/坏/中性三类 | ⬜ |
| B-2 | 探索事件池 | 20+ 探索事件：灵石矿、前辈指点、妖兽拦路、秘境入口、心魔劫… | ⬜ |
| B-3 | 奇遇系统 | 低概率高回报事件链：洞府传承、仙人赠法、转世重修。含 **小说奇遇事件**（5 类 14 个，见 `design-novel-events.md`） | ⬜ |
| B-4 | 日常事件 | 每"天"自动触发：集市折扣、门派邀请、天气影响修炼效率 | ⬜ |
| B-5 | 事件日志 | 分类日志面板（战斗/探索/奇遇/系统），颜色区分，可回溯 | ⬜ |

## Milestone C: 物品与经济 ⬜

> 目标：灵石有用处，丹药有讲究，装备有追求。

| Stage | 名称 | 说明 | 状态 |
|-------|------|------|------|
| C-1 | 背包系统 | 物品栏 UI，分类标签（丹药/材料/装备/杂物），堆叠 | ⬜ |
| C-2 | 丹药系统 v2 | 炼丹成功率受 `alchemy` 资质影响，丹药品质随机，念力消耗 | ⬜ |
| C-3 | 装备系统 | 武器/防具/饰品槽位，装备提供属性加成，品级（白→绿→蓝→紫→橙） | ⬜ |
| C-4 | 商店系统 | NPC 商人，价格受 `charisma` 影响，限时折扣（日常事件触发） | ⬜ |
| C-5 | 炼器系统 | 消耗矿石 + 灵石打造装备，`smithing` 资质影响品质 | ⬜ |

## Milestone D: 功法与技能 ⬜

> 目标：战斗不再只有平A，功法让 build 多样化。

| Stage | 名称 | 说明 | 状态 |
|-------|------|------|------|
| D-1 | 功法系统 | 功法数据表（名称/类型/学习门槛/效果），学习速度受悟性+功法资质影响 | ⬜ |
| D-2 | 技能战斗 | 战斗中可选择使用功法（消耗 MP），功法类型匹配武器资质加成 | ⬜ |
| D-3 | 被动效果 | 修炼功法提供永久被动加成（+攻/+防/+暴击等） | ⬜ |
| D-4 | 神通 | 高级技能，需对应灵根资质，元素伤害 + 元素抗性体系 | ⬜ |

## Milestone E: 世界与地图 ⬜

> 目标：不同地点有不同妖兽和资源，脚力/`speed` 属性有意义。

| Stage | 名称 | 说明 | 状态 |
|-------|------|------|------|
| E-1 | 地图系统 | 多区域地图（新手村/妖兽山/秘境/仙都…），`speed` 影响移动消耗 | ⬜ |
| E-2 | 区域事件 | 每个区域有独立事件池和妖兽表 | ⬜ |
| E-3 | 秘境探索 | 限时副本，多层推进，Boss 战 | ⬜ |
| E-4 | 风水采矿 | `fengshui` 发现隐藏点，`mining` 提高采集效率 | ⬜ |

## Milestone F: 社交与NPC ⬜

> 目标：世界有生气，`charisma` 属性发光。

| Stage | 名称 | 说明 | 状态 |
|-------|------|------|------|
| F-1 | NPC 系统 | NPC 数据表（名称/境界/性格/好感度），`charisma` 影响初始好感 | ⬜ |
| F-2 | 对话系统 | 简单分支对话，好感度解锁不同选项 | ⬜ |
| F-3 | 门派系统 | 加入门派获得功法/资源，门派任务 | ⬜ |
| F-4 | PvP 切磋 | 与 NPC 修士战斗，胜利获得声望 | ⬜ |

## Milestone G: 进阶机制 ⬜

> 目标：后期内容，让长期玩家有事做。

| Stage | 名称 | 说明 | 状态 |
|-------|------|------|------|
| G-1 | 渡劫系统 | 化神→渡劫需要通过天劫战（特殊 Boss），失败降境界或沦为散仙 | ⬜ |
| G-2 | 转世重修 | 大乘期可选择转世，保留部分修为，重新随机先天属性 | ⬜ |
| G-3 | 成就系统 | 收集成就解锁称号和永久加成 | ⬜ |
| G-4 | 排行榜 | 本地多存档排行，或 Azure Functions 实现在线排行 | ⬜ |
| G-5 | 仙道境界 | 大乘之后：散仙/地仙→天仙→金仙，飞升上界，新区域解锁 | ⬜ |
| G-6 | 洪荒终局 | 太乙金仙→大罗金仙→准圣→圣人（混元大罗金仙），参考《佛本是道》 | ⬜ |

## Milestone H: 部署与体验优化 ⬜

> 目标：上线运营品质。

| Stage | 名称 | 说明 | 状态 |
|-------|------|------|------|
| H-1 | Azure SWA 部署 | GitHub Actions CI/CD，自动部署到 Azure Static Web Apps | ⬜ |
| H-2 | PWA 支持 | Service Worker + manifest，可安装为桌面/手机 App | ⬜ |
| H-3 | 音效系统 | Web Audio API，背景音乐 + 操作音效（可选） | ⬜ |
| H-4 | 多存档 | 支持多个存档槽位，导入/导出存档为 JSON | ⬜ |
| H-5 | 新手引导 | 首次进入的教程引导流程 | ⬜ |
