# T0054 — 历练悬赏任务

- **状态**: ⬜ 未开始
- **分类**: 世界与地图
- **前置**: T0021, T0025
- **Spec**: —

## 描述

参考鬼谷八荒的悬赏任务系统，实现玩家可接取的各类任务——讨伐妖兽、采集材料、护送NPC、探索秘境等。为游戏提供目标导向的玩法，而非纯随机探索。

## 核心需求

### 1. 悬赏榜

- 各区域城市的"悬赏榜"面板，展示当前可接取的任务
- 任务定期刷新（每月刷新部分任务）
- 任务有等级限制（境界要求）和时间限制（超时自动失败）
- 同时最多接取 3~5 个任务

### 2. 任务类型

| 类型 | 描述 | 目标 | 奖励 |
|------|------|------|------|
| 讨伐 | 消灭指定数量/种类的妖兽 | 击杀 X 只 Y 妖兽 | 灵石 + 材料 |
| 采集 | 收集指定材料 | 获取 X 个 Y 材料 | 灵石 + 丹药 |
| 护送 | 护送 NPC 到指定区域 | 安全抵达目的地 | 灵石 + 好感度 |
| 探索 | 探索指定秘境/区域 | 到达并返回 | 灵石 + 稀有物品 |
| 传信 | 将信件送到指定 NPC | 找到 NPC 交付 | 灵石 + NPC好感度 |
| 炼制 | 为 NPC 炼制指定丹药/装备 | 提交成品 | 灵石 + 材料 + 配方 |
| 切磋 | 击败指定 NPC（不致死） | 战斗胜利 | 灵石 + 功法线索 |

### 3. 任务品质

| 品质 | 颜色 | 奖励倍率 | 难度 |
|------|------|----------|------|
| 普通 | 白 | 1× | 低 |
| 精英 | 绿 | 2× | 中 |
| 困难 | 蓝 | 3× | 高 |
| 传说 | 紫 | 5× | 极高 |
| 仙级 | 金 | 10× | 地狱 |

### 4. 任务数据结构

```ts
interface QuestDef {
  id: string;                  // 'core:quest_slay_wolf_1'
  type: 'slay' | 'gather' | 'escort' | 'explore' | 'deliver' | 'craft' | 'duel';
  name: string;                // '讨伐狂暴野狼'
  description: string;
  quality: Quality;
  minRealm: number;            // 最低境界
  timeLimit: number;           // 时限（月数）
  region?: string;             // 限定区域
  objectives: QuestObjective[];// 目标列表
  rewards: QuestReward[];      // 奖励列表
  weight: number;              // 刷新权重
}

interface QuestObjective {
  type: 'kill' | 'collect' | 'reach' | 'deliver' | 'craft' | 'win_combat';
  targetId: string;            // 目标 ID（妖兽/物品/NPC/区域）
  count: number;               // 需要数量
  current?: number;            // 当前进度
}
```

### 5. 任务追踪

- 已接取任务显示在 HUD 或专用面板中
- 实时显示任务进度（如 3/5 只妖兽）
- 任务完成后自动提示，可选择"交付"获取奖励
- 任务超时或放弃：扣少量声望

### 6. 门派任务（与 T0027 联动）

- 加入门派后可接取门派专属任务
- 完成门派任务获得贡献点（换取功法/资源）
- 门派任务难度更高但奖励更丰厚

### 7. 通过 DLC 注册

- 任务定义通过 `DLCPack.quests?: QuestDef[]` 注册
- 任务目标和奖励引用全局注册表中的物品/妖兽/NPC

## 参考

- 鬼谷八荒：各城市悬赏榜可接任务，门派有专属任务线
- 任务是获得灵石和声望的稳定渠道
