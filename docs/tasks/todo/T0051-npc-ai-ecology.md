# T0051 — NPC AI 生态

- **状态**: ⬜ 未开始
- **分类**: 社交与NPC
- **前置**: T0025, T0021
- **Spec**: —

## 描述

参考鬼谷八荒的核心体验——"世界在自行运转"。NPC 不仅是静态数据，而是有自主行为的活跃实体：自动修炼、突破、战斗、死亡、结婚，让玩家感受到"活的修仙世界"。

## 核心需求

### 1. NPC 生命周期

- 每个 NPC 有完整的修仙生命周期：
  - 出生（或初始存在）→ 修炼 → 突破 → 渡劫 → 飞升/陨落
  - NPC 有寿命，寿命耗尽则自然死亡
  - NPC 可在战斗和渡劫中死亡
- 世界定期刷新新的 NPC（新生代修士），保持世界活力
- NPC 死亡后从世界中移除，但留有"墓碑"记录（可在日志/地图中查看）

### 2. NPC 自主行为（每回合/每月 tick）

每个时间推进周期（月），所有活跃 NPC 执行行为决策：

| 行为 | 权重 | 效果 |
|------|------|------|
| 修炼 | 高 | 增加修为，缓慢突破 |
| 突破 | 中（达到瓶颈时） | 尝试突破，成功/失败 |
| 探索 | 中 | 获取物品/奇遇 |
| 战斗 | 低 | 与其他 NPC 或妖兽交战 |
| 交易 | 低 | NPC 间买卖物品 |
| 结交 | 低 | NPC 间建立关系 |

- NPC 行为受性格影响：
  - 好战型：战斗权重大幅提高
  - 苟道型：修炼权重高，回避战斗
  - 社交型：交易和结交权重高

### 3. NPC 势力关系

- NPC 之间有关系网：友好/中立/敌对
- 同门派 NPC 默认友好
- NPC 间可因争夺资源/秘境而产生冲突
- NPC 关系变化会产生世界事件（如"XX 与 YY 大打出手"）

### 4. 世界事件广播

- NPC 的重要行为生成世界事件，显示在日志中：
  - "散修张三成功突破金丹期！"
  - "天骄李四于秘境中陨落…"
  - "王五与赵六结为道侣。"
  - "落叶宗宗主闭关渡劫中…"
- 世界事件按重要性分级：普通（灰色）/ 重要（白色）/ 重大（金色）

### 5. NPC 实力成长

- NPC 修炼速度基于其灵根资质和功法
- 天才 NPC 成长更快，平庸 NPC 可能卡在低境界一辈子
- NPC 境界分布大致符合金字塔结构：
  - 练气期 NPC 占 60%
  - 筑基期占 25%
  - 金丹期占 10%
  - 元婴+ 占 5%

### 6. 性能考量

- NPC 行为计算采用简化模型（不走完整战斗流程，用概率判定结果）
- 活跃 NPC 数量上限（如 100~200），超出后低境界 NPC 自动"隐退"
- NPC 行为每月 batch 处理，避免逐个 tick 导致性能问题

## 数据结构

```ts
interface NPCState {
  id: string;
  name: string;
  realm: number;           // 当前境界
  cultivation: number;     // 当前修为
  lifespan: number;        // 剩余寿命
  personality: NPCPersonality;
  sect?: string;           // 所属门派
  spouse?: string;         // 道侣 NPC ID
  relationships: Map<string, number>; // 与其他 NPC 的好感度
  inventory: string[];     // 持有物品
  techniques: string[];    // 已学功法
  isAlive: boolean;
}
```
